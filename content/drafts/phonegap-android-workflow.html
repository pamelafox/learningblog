Mobile development takes more time than web development, because of the additional Since mobile development involves more When I'm developing the website for everyday.io, I can just spin up the website in my browser, make a change, and reload. When I'm developing the mobile Android app, I actually have to compile the APK and upload it to my device to test if my changes work. It's a slower process (and I don't have time for slow processes!), so I've done what I can to optimize the workflow. 1) Test in browser My Phonegap code starts off as a view in my website, which means I can just enter "http://test.everyday.io/mobile" and see how my code works.
